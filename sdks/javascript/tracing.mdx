# Distributed Tracing

The Foil JavaScript SDK provides comprehensive distributed tracing for complex AI applications. This guide covers advanced tracing patterns for multi-step agent workflows.

<Tip>
  **Full examples**: [customer-support-agent](https://github.com/getfoil/foil-examples/tree/main/customer-support-agent) (SDK tracing with tools) | [rag-agent](https://github.com/getfoil/foil-examples/tree/main/rag-agent) (retriever spans and LLM synthesis)
</Tip>

## Trace Architecture

A **trace** represents a complete request or workflow. Each trace contains multiple **spans** organized in a tree structure:

```
Trace: handle-user-request
├── Span: agent (root)
│   ├── Span: llm (gpt-4o) — plan generation
│   │   ├── Span: tool (web-search)
│   │   └── Span: tool (calculator)
│   └── Span: llm (gpt-4o) — final response
```

Every span has:
- **traceId** - Links all spans in the same trace
- **spanId** - Unique identifier for this span
- **parentSpanId** - Links to parent span (if not root)
- **depth** - Nesting level (0 for root)

## Creating Traces

```javascript
import { Foil } from '@getfoil/foil-js';

const foil = new Foil({
  apiKey: process.env.FOIL_API_KEY,
  agentName: 'my-agent',
});

const result = await foil.trace(async (ctx) => {
  // ctx is TraceContext with traceId already set
  console.log('Trace ID:', ctx.traceId);

  // All spans created here share this traceId
  return await processRequest(ctx);
}, {
  name: 'process-request',  // Optional trace name
  traceId: 'custom-id',     // Optional: provide your own trace ID
  sessionId: 'conv-123',    // Optional: conversation/session ID
  input: userQuery,          // Optional: trace-level input
  properties: {              // Optional: custom metadata
    userId: 'user-123',
  },
});
```

## Managing Span Hierarchy

### Automatic Nesting with Convenience Methods

Use `ctx.llmCall()`, `ctx.tool()`, and other convenience methods for automatic span management:

```javascript
await foil.trace(async (ctx) => {
  // LLM call span
  const response = await ctx.llmCall('gpt-4o', async () => {
    return await openai.chat.completions.create({
      model: 'gpt-4o',
      messages,
    });
  });

  // Tool spans nest under the trace
  const result1 = await ctx.tool('search', async () => {
    return await searchAPI(query);
  });

  const result2 = await ctx.tool('calculator', async () => {
    return compute(result1);
  });

  // Second LLM call
  const finalResponse = await ctx.llmCall('gpt-4o', async () => {
    return await openai.chat.completions.create({
      model: 'gpt-4o',
      messages: [...messages, { role: 'user', content: result2 }],
    });
  });
});
```

### Manual Span Control

For advanced flows where you need explicit control over parent-child relationships:

```javascript
await foil.trace(async (ctx) => {
  const rootSpan = await ctx.startSpan(SpanKind.AGENT, 'root');

  // Create child context for parallel work
  const childCtx = rootSpan.createChildContext();

  // This span is explicitly a child of rootSpan
  const childSpan = await childCtx.startSpan(SpanKind.LLM, 'gpt-4o');
  await childSpan.end({ output: '...' });

  await rootSpan.end({ output: 'done' });
});
```

## Parallel Span Execution

Track parallel operations correctly:

```javascript
await foil.trace(async (ctx) => {
  // Plan which tools to run
  const planResponse = await ctx.llmCall('gpt-4o', async () => {
    return await openai.chat.completions.create({
      model: 'gpt-4o',
      messages: [{ role: 'user', content: 'What tools do I need?' }],
    });
  });

  const tools = ['search', 'calculator', 'weather'];

  // Execute tools in parallel
  const results = await Promise.all(
    tools.map(async (toolName) => {
      return await ctx.tool(toolName, async () => {
        return await executeTool(toolName);
      });
    })
  );

  // Final synthesis
  const finalResponse = await ctx.llmCall('gpt-4o', async () => {
    return await openai.chat.completions.create({
      model: 'gpt-4o',
      messages: [{ role: 'user', content: `Synthesize: ${JSON.stringify(results)}` }],
    });
  });
});
```

## OpenAI Function Calling with executeTools

The `ctx.executeTools()` helper automatically executes and traces tool calls from an OpenAI response:

```javascript
await foil.trace(async (ctx) => {
  const messages = [{ role: 'user', content: 'What is the weather in Paris?' }];
  const tools = [{
    type: 'function',
    function: {
      name: 'get_weather',
      parameters: {
        type: 'object',
        properties: { location: { type: 'string' } },
      },
    },
  }];

  // First LLM call — may request tool calls
  const response = await ctx.llmCall('gpt-4o', async () => {
    return await openai.chat.completions.create({
      model: 'gpt-4o',
      messages,
      tools,
    });
  });

  // Execute tool calls — each gets its own traced tool span
  const toolMessages = await ctx.executeTools(response, {
    get_weather: async (args) => {
      return { temp: 22, condition: 'sunny', location: args.location };
    },
  });

  // Continue conversation with tool results
  messages.push(response.choices[0].message, ...toolMessages);

  const finalResponse = await ctx.llmCall('gpt-4o', async () => {
    return await openai.chat.completions.create({
      model: 'gpt-4o',
      messages,
    });
  });

  return finalResponse.choices[0].message.content;
}, { name: 'weather-agent' });
```

`executeTools()` returns an array of `{ role: 'tool', tool_call_id, content }` messages ready to append to your conversation.

## Tool Execution Patterns

### Simple Tool Wrapping

```javascript
const result = await ctx.tool('search', async () => {
  return await searchAPI(query);
}, {
  input: { query },
  properties: { source: 'google' },
});
```

### Tool with Error Handling

```javascript
const result = await ctx.tool('database-query', async () => {
  try {
    return await db.query(sql);
  } catch (error) {
    // Error automatically captured in span
    throw error;
  }
}, {
  input: { sql },
});
```

## Multi-Agent Workflows

Track interactions between multiple agents:

```javascript
const foil = new Foil({
  apiKey: process.env.FOIL_API_KEY,
  agentName: 'orchestrator',
});

await foil.trace(async (ctx) => {
  // Orchestrator decides which agent to call
  const planResponse = await ctx.llmCall('gpt-4o', async () => {
    return await openai.chat.completions.create({
      model: 'gpt-4o',
      messages: [{ role: 'user', content: 'Research topic X' }],
    });
  });

  // Delegate to research agent (nested spans)
  const researchResults = await ctx.tool('research-agent', async () => {
    // Research agent does its own work
    const searchResults = await ctx.tool('web-search', async () => {
      return await webSearch(query);
    });

    const analysis = await ctx.llmCall('gpt-4o', async () => {
      return await openai.chat.completions.create({
        model: 'gpt-4o',
        messages: [{ role: 'user', content: `Analyze: ${searchResults}` }],
      });
    });

    return analysis.choices[0].message.content;
  });

  return researchResults;
});
```

## Retrieving Traces

Fetch completed traces for debugging or analysis:

```javascript
// Get a specific trace with all spans
const trace = await foil.getTrace(traceId);

console.log('Trace:', trace.traceId);
console.log('Spans:', trace.spans.length);

for (const span of trace.spans) {
  console.log(`${span.depth}: ${span.spanKind} - ${span.name}`);
  console.log(`  Duration: ${span.durationMs}ms`);
  console.log(`  Tokens: ${span.tokens?.total || 'N/A'}`);
}
```

## Trace Context Propagation

Pass trace context to external services:

```javascript
await foil.trace(async (ctx) => {
  // Pass context to external service via headers
  const response = await fetch('https://another-service.com/api', {
    headers: {
      'x-trace-id': ctx.traceId,
      'x-parent-span-id': ctx.currentParentEventId,
    },
  });

  // Continue with response...
});
```

## Performance Tips

### Batch Span Operations

For high-throughput scenarios, minimize API calls:

```javascript
// Good: Use convenience methods for automatic management
await ctx.llmCall('gpt-4o', async () => {
  return await processBatch(items);
});

// Avoid: Creating many small spans in tight loops
for (const item of items) {
  // This creates too many API calls
  await ctx.startSpan(SpanKind.CUSTOM, 'process-item');
}
```

### Fire-and-Forget Logging

For non-critical telemetry, use the low-level logging API:

```javascript
import { Foil } from '@getfoil/foil-js';

const foil = new Foil({ apiKey: process.env.FOIL_API_KEY });

// Fire and forget - doesn't wait for response
foil.log({
  model: 'gpt-4o',
  input: messages,
  output: response,
  latency: duration,
});
```
